# Промпт для генерации фронтенда KYDiR

> Скопируй всё, что ниже, и отдай ИИ, которая будет делать фронтенд.

---

Сделай фронтенд для веб-приложения **KYDiR — общий семейный бюджет**. Несколько членов семьи ведут один общий бюджет: записывают траты и доходы, смотрят аналитику по категориям и по людям.

Бэкенд уже готов (FastAPI) и раздаёт фронтенд как статику. Твоя задача — только фронтенд.

## Формат поставки

- **Статические файлы** в папке `frontend/`: `index.html` + CSS/JS рядом. Точка входа — `frontend/index.html`.
- **Без шага сборки.** Никаких npm/webpack/vite. Разрешены библиотеки с CDN (например, Vue 3 или Preact через `<script>`, Chart.js для графиков). Можно и на чистом JS — на твой вкус.
- SPA: всё приложение на одной странице, переключение вкладок без перезагрузки.
- Язык интерфейса — русский. Валюта — рубли, формат `12 345,67 ₽`.

## Интеграция с API

- Все запросы — `fetch` на относительные пути `/api/...` (фронт и API на одном домене).
- Авторизация через **httpOnly-cookie**: используй `credentials: 'same-origin'` во всех fetch. Никаких токенов в localStorage.
- Любой ответ **401** → показать экран входа.
- Ошибки приходят как `{"detail": "текст на русском"}` — показывай их пользователю (тост/инлайн).
- Даты в API — строки `YYYY-MM-DD`. Суммы — числа в рублях (`1234.56`).

## API-контракт

### Авторизация
| Метод | Путь | Тело / параметры | Ответ |
|---|---|---|---|
| POST | `/api/auth/login` | `{"username": "papa", "password": "..."}` | `200` → объект пользователя, ставит cookie; `401` при неверных данных |
| POST | `/api/auth/logout` | — | `204` |
| GET | `/api/auth/me` | — | текущий пользователь или `401` |

Объект пользователя: `{"id": 1, "username": "papa", "display_name": "Папа", "color": "#4f8ef7"}` — `color` используй для подписи автора операции и на графике «кто потратил».

### Пользователи (члены семьи)
| Метод | Путь | Тело | Ответ |
|---|---|---|---|
| GET | `/api/users` | — | массив пользователей |
| POST | `/api/users` | `{"username", "display_name", "password", "color"}` | `201`; `409` если username занят |
| PATCH | `/api/users/me` | любые из `{"display_name", "color", "password"}` | обновлённый пользователь |

Открытой регистрации нет — новых членов семьи добавляют из настроек.

### Категории
Объект: `{"id": 3, "name": "Продукты", "type": "expense", "icon": "🛒", "color": "#4caf50", "archived": false}`. `type` — `"expense"` или `"income"`, `icon` — эмодзи.

| Метод | Путь | Тело / параметры | Ответ |
|---|---|---|---|
| GET | `/api/categories?type=expense&include_archived=false` | оба параметра опциональны | массив категорий |
| POST | `/api/categories` | `{"name", "type", "icon", "color"}` | `201` |
| PATCH | `/api/categories/{id}` | любые из `{"name", "icon", "color", "archived"}` | обновлённая категория |
| DELETE | `/api/categories/{id}` | — | `204`; `409` если у категории есть операции (тогда предлагай архивировать) |

### Операции (траты и доходы)
| Метод | Путь | Тело / параметры |
|---|---|---|
| GET | `/api/transactions` | параметры: `date_from`, `date_to`, `category_id`, `user_id`, `type`, `q` (поиск по комментарию), `page` (с 1), `per_page` (по умолч. 50) |
| POST | `/api/transactions` | `{"amount": 1234.56, "type": "expense", "category_id": 3, "date": "2026-07-15", "comment": "Пятёрочка"}` → `201` |
| PATCH | `/api/transactions/{id}` | любые из `{"amount", "category_id", "date", "comment"}` |
| DELETE | `/api/transactions/{id}` | → `204` |

Ответ списка (пагинация):
```json
{
  "items": [
    {
      "id": 1, "amount": 1234.56, "type": "expense",
      "date": "2026-07-15", "comment": "Пятёрочка",
      "category": {"id": 3, "name": "Продукты", "type": "expense", "icon": "🛒", "color": "#4caf50", "archived": false},
      "user": {"id": 1, "username": "papa", "display_name": "Папа", "color": "#4f8ef7"}
    }
  ],
  "total": 120, "page": 1, "per_page": 50, "pages": 3
}
```
Автор (`user`) проставляется бэкендом автоматически — в форме добавления его выбирать не нужно.

### Аналитика
`GET /api/analytics/summary?date_from=2026-07-01&date_to=2026-07-31` (без параметров — текущий месяц):
```json
{
  "date_from": "2026-07-01", "date_to": "2026-07-31",
  "income_total": 100000.0, "expense_total": 45230.5, "balance": 54769.5,
  "expense_by_category": [
    {"category_id": 3, "name": "Продукты", "icon": "🛒", "color": "#4caf50", "total": 18400.0, "count": 24}
  ],
  "income_by_category": [ /* та же структура */ ],
  "expense_by_user": [
    {"user_id": 1, "display_name": "Папа", "color": "#4f8ef7", "total": 21000.0}
  ]
}
```

`GET /api/analytics/monthly?months=12` — для графика динамики:
```json
[ {"month": "2026-06", "income": 95000.0, "expense": 51200.0}, {"month": "2026-07", "income": 100000.0, "expense": 45230.5} ]
```
Месяцы идут подряд без пропусков (пустые = нули).

Для сравнения периодов («в этом месяце на еду на 20% больше») вызывай `summary` дважды с разными периодами и сравнивай на клиенте.

## Экраны

1. **Вход** — имя + пароль, по центру, показ ошибки при 401.
2. **Дашборд** (главная):
   - переключатель периода: текущий месяц (по умолчанию) / прошлый месяц / 3 месяца / год / произвольный;
   - три крупные цифры: доходы, расходы, остаток;
   - пончик расходов по категориям (цвета и иконки из данных);
   - блок «кто потратил» по членам семьи (их цвета);
   - сравнение с прошлым периодом по топ-категориям (стрелки ↑↓ и %);
   - линейный/столбчатый график динамики по месяцам (`/api/analytics/monthly`);
   - последние 5–10 операций.
3. **Операции** — список с фильтрами (период, категория, человек, тип, поиск), группировка по дням, пагинация или «показать ещё», редактирование и удаление (с подтверждением).
4. **Категории** — два списка (траты/доходы), создание, правка (имя/эмодзи/цвет), архивация; при 409 на удалении — предложить архивировать.
5. **Настройки** — мой профиль (имя, цвет, смена пароля), добавление члена семьи, кнопка выхода.

## Ключевой UX — быстрое добавление операции

Плавающая кнопка «+» на всех экранах. По нажатию — модалка/шторка:
- крупное поле суммы с цифровой клавиатурой (`inputmode="decimal"`),
- переключатель Трата/Доход,
- сетка категорий (иконка + название), частые — первыми,
- дата по умолчанию сегодня, комментарий опционален,
- сохранение в 2–3 касания. Это главный сценарий использования — 90% визитов будут «внести трату с телефона».

## Дизайн

- **Mobile-first**: телефон — основной сценарий; на десктопе — адаптив с боковой навигацией, на мобиле — нижний таб-бар.
- Тёмная и светлая тема по `prefers-color-scheme`.
- Аккуратный современный минимализм; расходы — красноватый акцент, доходы — зелёный.
- Скелетоны/спиннеры на загрузку, пустые состояния с подсказками («Пока нет операций — нажмите +»).

## Проверка

Для локальной проверки бэкенд запускается так (из корня репозитория):
```
pip install -r requirements.txt
python manage.py init-db && python manage.py seed-categories
python manage.py add-user --username test --name "Тест" --password test1234
uvicorn app.main:app --reload
```
После этого твой `frontend/` доступен на `http://localhost:8000/`, API — на `/api`, интерактивная документация — на `/api/docs`.
